Подробное руководство по Nikto

Nikto - это сканер веб-сервера с открытым исходным кодом, который сканирует целевой веб-сервер на наличие различных уязвимостей и проблем конфигурации.

Что такое Nikto?

Nikto - это инструмент командной строки, который выполняет всестороннее тестирование веб-сервера, включая:

  • Обнаружение небезопасных файлов и каталогов.
  • Выявление устаревшего программного обеспечения и известных уязвимостей.
  • Проверку параметров конфигурации сервера.
  • Идентификацию установленных приложений и их версий.

Как работает Nikto?

Nikto работает, отправляя серию запросов к целевому веб-серверу и анализируя ответы. Он использует базу данных известных уязвимостей и шаблонов для выявления потенциальных проблем.

Основные возможности Nikto

Nikto предлагает широкий спектр возможностей, включая:

  • Поддержку протоколов HTTP и HTTPS.
  • Возможность сканирования нескольких портов на одном сервере.
  • Обнаружение различных типов уязвимостей, включая SQL-инъекции, XSS и другие.
  • Идентификацию устаревших серверов и программного обеспечения.
  • Настраиваемый вывод в различных форматах.
  • Обход систем обнаружения вторжений (IDS).

Примеры использования Nikto

Базовое сканирование

Этот пример демонстрирует базовое сканирование веб-сервера. 


# Выполнение базового сканирования:
nikto -h http://example.com

# Что вы увидите в терминале:
# Nikto начнет сканирование http://example.com и отобразит результаты в режиме реального времени.
# Пример вывода:
# + Target IP:          192.0.2.1
# + Target Hostname:    example.com
# + Target Port:        80
# + Start Time:         2024-07-24 12:00:00
# ...
# + Server: Apache/2.4.29 (Ubuntu)
# + The anti-clickjacking X-Frame-Options header is not present.
# + The X-XSS-Protection header is not defined. This can allow the browser to render pages susceptible to XSS attacks in a non-standards compliant mode.
# + /robots.txt file found: Disallow: /admin/
# + /cgi-bin/ доступен: может содержать уязвимые скрипты.
# + Server leaks inodes via ETags, header found.
# + ...
# + 10080 items checked: 0 found 7 vulnerabilities (0.00%) 0 item(s) reported on remote host
# + End Time:           2024-07-24 12:30:00
# + 1 host scanned.

Сканирование с указанием порта

Этот пример показывает, как указать порт для сканирования. 


# Сканирование порта 8080:
nikto -h http://example.com -p 8080

# Что вы увидите в терминале:
# Nikto будет сканировать веб-сервер example.com на порту 8080.
# Если на этом порту работает веб-сервер, Nikto отобразит результаты сканирования.

Сохранение результатов в файл

Этот пример демонстрирует, как сохранить результаты сканирования в файл. 


# Сохранение результатов в формате HTML:
nikto -h http://example.com -o report.html

# Что вы увидите:
# Nikto выполнит сканирование и сохранит отчет в файл "report.html" в текущей директории.
# Файл будет содержать подробные результаты сканирования в формате HTML.

Обход IDS

Этот пример показывает, как использовать опции для обхода систем обнаружения вторжений (IDS). 


# Использование случайной задержки и поддельных заголовков для обхода IDS:
nikto -h http://example.com -T 3 -id myuser -pw mypass -p 80 --randomise

# Что вы увидите в терминале:
# Nikto будет отправлять запросы с небольшой случайной задержкой между ними.
# Nikto будет использовать указанные имя пользователя и пароль для аутентификации (если это необходимо).
# Nikto добавит случайные параметры к каждому запросу, чтобы затруднить обнаружение сканирования.

Важные замечания

  • Используйте Nikto только на тех системах, которые вам разрешено тестировать. Несанкционированное использование Nikto может привести к юридическим последствиям.
  • Будьте осторожны при сканировании веб-серверов. Чрезмерное сканирование может привести к блокировке вашего IP-адреса.
  • Всегда проверяйте результаты сканирования вручную, чтобы избежать ложных срабатываний.
